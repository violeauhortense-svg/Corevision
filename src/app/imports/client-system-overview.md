1️⃣ Fiche client – cœur du système
Tout part de la fiche client. C’est le centre névralgique.
Tous les modules s’y connectent ou s’en alimentent.
Contenus :
•	Informations générales (nom, société, fiscalité, statut, etc.)
•	Bandeau synthèse (KPI, score de suivi, recommandations actives, actions en attente)
•	Accès direct aux onglets : Foyer, Revenus & Imposition, Patrimoine, Objectifs, Tâches, Documents, Audit, Historique
Rôle central :
•	Identifie le client
•	Sert de point d’ancrage pour tous les flux : tâches, recommandations, interactions, mises en place, documents
________________________________________
2️⃣ Timeline / Historique
Lien principal : Fiche client → Historique
•	Chaque interaction (mail, RDV, appel, document envoyé, recommandation proposée/validée) est enregistrée ici.
•	Les événements viennent de tous les autres modules :
o	RDV → créé dans l’onglet Tâches ou directement depuis Audit
o	Emails → ajoutés via l’onglet Documents ou synchronisation automatique
o	Recommandations → changent de statut (Proposée → Validée → Mise en place)
•	Objectif : visualiser la chronologie complète de la relation client
Flux :
•	Nouvelle interaction → Historique mis à jour → Bandeau synthèse (score de suivi et actions en attente recalculés)
________________________________________
3️⃣ Module Audit / Recommandations
Lien principal : Fiche client ↔ Audit
•	Chaque recommandation est créée depuis Audit
•	Contient : contexte, problématique, solution, avantages, risques, actions associées
•	Liée à :
o	Tâches (chaque action peut générer une tâche)
o	Historique (validation ou mise à jour enregistrée comme interaction)
o	Bandeau synthèse (nombre de recommandations actives et stratégies mises en place)
Workflow type :
1.	Recommandation créée → statut “À analyser”
2.	Conseiller analyse → statut “Proposée”
3.	Client valide → statut “Validée”
4.	Mise en place commence → workflow dans SUIVI DE MISE EN PLACE
________________________________________
4️⃣ Module Suivi de mise en place
Lien principal : Audit → Suivi de mise en place
•	Chaque recommandation validée peut générer un workflow visuel de mise en place
•	Étapes : Recommandation validée → Simulation → Dossier envoyé → Signature → Transmission → Contrat actif
•	Chaque étape peut déclencher :
o	une tâche pour toi ou un partenaire
o	une mise à jour dans l’historique
o	recalcul du score de suivi client
Impact :
•	Permet de voir en un coup d’œil si une recommandation est complètement mise en place ou bloquée
•	Centralise la responsabilité de suivi
________________________________________
5️⃣ Onglet Patrimoine
Lien principal : Fiche client → Patrimoine
•	Sépare patrimoine personnel et professionnel
•	Chaque patrimoine peut influencer :
o	les recommandations (ex : optimisation fiscale, investissement)
o	les tâches (ex : création d’une holding, arbitrage immobilier)
•	Les flux financiers (rémunération/dividendes) peuvent alimenter :
o	Score de suivi (indicateur de liquidité ou opportunité)
o	Analyse de recommandations (ex : PER, assurance-vie)
________________________________________
6️⃣ Onglet Objectifs / Plan stratégique
Lien principal : Fiche client → Objectifs → Audit et Tâches
•	Chaque objectif est lié à une ou plusieurs recommandations
•	Permet de visualiser la roadmap patrimoniale du client
•	Les actions associées aux objectifs peuvent générer des tâches et influencer le workflow de mise en place
________________________________________
7️⃣ Onglet Tâches
Lien principal : Tâches ↔ Recommandations / RDV / Objectifs
•	Chaque tâche a un responsable, une échéance et un statut
•	Les tâches peuvent être générées automatiquement à partir :
o	d’un RDV (ex : “Envoyer dossier PER”)
o	d’une recommandation validée (ex : “Créer holding”)
o	d’un objectif (ex : “Préparer transmission”)
•	Mise à jour des tâches → recalcul automatique du score de suivi client et bandeau KPI
________________________________________
8️⃣ Bandeau synthèse / Dashboard client
Lien principal : Fiche client ↔ tous les modules
•	Agrège les KPI et indique :
o	Score de suivi global
o	Nombre de recommandations actives
o	Stratégies mises en place
o	Actions en attente
•	Reflète en temps réel les mises à jour des modules : tâches, recommandations, mises en place, historique
________________________________________
9️⃣ Pipeline
Lien principal : Pipeline ↔ Fiche client / Stratégies
•	Reflète le cycle de conseil patrimonial
o	Prospect → Découverte → Audit → Stratégie définie → Recommandations → Mise en place → Suivi
•	Chaque étape est liée à :
o	Recommandations associées
o	Tâches à réaliser
o	Historique d’interactions
Objectif : voir l’état global de ton portefeuille client et les prochaines actions prioritaires
________________________________________
🔗 Résumé visuel des flux
Fiche client
│
├─ Bandeau synthèse (KPI)
├─ Historique (timeline des interactions)
├─ Patrimoine (perso + pro)
├─ Objectifs → Recommandations
├─ Audit → workflow mise en place
├─ Tâches (liées aux objectifs, recommandations, RDV)
└─ Documents (attache tout)
 
Pipeline ↔ Fiche client / Recommandations
Flux clés :
•	Une recommandation validée → génère workflow mise en place → crée des tâches → mise à jour historique → mise à jour bandeau KPI
•	Un RDV ou mail → enregistre dans historique → peut générer tâches → influence score de suivi
•	Les objectifs → agrègent les recommandations → génèrent des tâches → reflètent la roadmap dans le dashboard
